# 로또 랩(Lotto Lab) 개발 일지 및 아키텍처 정리

본 문서는 AI Agent가 사용자와 협력하여 `lotto-lab` 프로젝트를 기획, 디자인, 구현 및 점검해 나간 전체 개발 과정을 기록한 문서입니다.

## 1. 프로젝트 기획 및 초기 아키텍처 설계
* **목표 설정:** 단순한 랜덤 번호 생성이 아닌, 한국 동행복권 6/45 역대 당첨 데이터 기반의 통계적 분석을 통해 번호를 추천하는 웹 애플리케이션 개발.
* **기술 스택 선정:** 
  * 프레임워크: Next.js (App Router, React 19)
  * 스타일링: Tailwind CSS v4
  * 데이터: 로컬 정적 JSON 파일 (`public/data/lotto_history.json`)
* **핵심 알고리즘 3종 기획:**
  1. **흐름 추적형 (Bayesian):** 번호별 누적 출현 빈도를 가중치로 삼아 비복원 추출.
  2. **균형 필터형 (Constraint):** 당첨 번호 합계 연속성, 홀짝 비율 등 역대 1등 당첨 데이터의 통계적 황금 비율(필터)을 만족하는 번호 추출.
  3. **패턴 매칭형 (Pattern):** 번호 간의 분포 간격(Gap)을 분석하고 노이즈를 섞어 과거 당첨 배열과 유사한 패턴의 번호 추출.

## 3. 데이터 파이프라인 구축 (실제 데이터 확보)
* **문제 발생:** 동행복권 OpenAPI 서버가 봇 접근을 차단하여, 앱 내 직접 Fetching이 실패함.
* **해결 및 파이프라인 정립 (`fetch_lotto_data.py`):**
  * 오픈소스 GitHub SQLite DB(`happylie/lotto_data`)를 활용하여 최신 100회차 구간의 데이터를 파이썬 스크립트로 다운로드 및 파싱.
  * DB에 아직 반영되지 않은 최신 회차(1205회~1216회)는 `add_manual_data.py`를 작성하여 수동 주입을 자동화.
  * 총 150회차 분량의 정확한 최신 당첨 통계 데이터를 JSON 형태로 시스템에 병합 성공.

## 4. 알고리즘 엔진 검증 및 고도화 (QA)
TypeScript로 작성된 엔진들을 파이썬 분석 스크립트(`verify_engines.py`)로 10,000회 이상 교차 시뮬레이션하여 논리적 결함을 발견하고 고도화함.

* **Bayesian (베이지안) 엔진 고도화:** 원래는 전체 회차 데이터를 동일한 비중으로 다뤘으나, *'최근 50회차 데이터에 1.5배의 가중치 보너스'*를 주도록 수정하여 최신 트렌드를 잘 추적하도록 개선.
* **Constraint (필터형) 엔진 버그 수정:** 기존의 연속 번호 카운트 로직이 '우연히만' 작동하던 문제를 파악하고, 명시적으로 `hasTripleConsecutive`(3연속 번호)를 차단하는 헬퍼 함수를 추가. 합리적인 폴백(Fallback) 처리 추가.
* **Pattern (패턴형) 엔진 다양성 확보:** 번호 간격의 노이즈 허용 범위를 `-2~+2`에서 `-3~+3`으로 넓히고, 시작 숫자의 범위도 확장하여 번호 쏠림 현상을 방지하고 다양한 번호대가 출력되도록 보완.
